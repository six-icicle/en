# Project Context тАФ En

## What this is
**En** (ч╕Б тАФ connection, fate, the central point that links) is a desktop hub for managing many concurrent Claude CLI sessions on Mac. Tauri 2 app (Rust core + React/TypeScript webview frontend) with embedded terminals via xterm.js. Single hub window: themed grid of terminal sessions, every tile visible at once. *Not* a generic terminal app тАФ purpose-built for running many `claude` instances side-by-side and knowing which need attention. Renamed from "multiclaude" 2026-05-20. Repo: `git@github.com:six-icicle/en.git`. Bundle ID: `dev.sixicicle.en`. Crate: `en` / `en_lib`.

## Why
User runs many `claude` sessions per day across different projects and wants one place to navigate, organize, and surface which sessions need input тАФ instead of cycling through dozens of iTerm2 windows.

## Stack
- Frontend: React 19 + TypeScript + shadcn/ui + Vite + xterm.js (WebGL addon)
- Backend: Tauri 2 + Rust + portable-pty + tiny_http (status-detection HTTP receiver)
- Storage: SQLite via rusqlite (planned тАФ metadata only; sessions are ephemeral, die on hub quit)
- Aesthetic: tea-ceremony minimalism, "lifted" lifted-bg recipe (the canonical en feel). **Brand wordmark: Outfit 600/16px**, body sans **Space Grotesk 500** (swapped from Inter), mono JetBrains Mono. **Default theme: rose-pine-soft + seigaiha texture**. **8 themes** (4 families ├Ч 2 variants) + 14 textures + accent/bg pickers + texture-amount and font-size scales. Full design law captured in `mockups/49-brand-blueprint.html` (visual + copy-pasteable JSON for AI hand-off).
